# Add GET/PUT /settings API endpoint

Register GET /settings and PUT /settings on the HTTP server (unconditionally, not behind any flag). Both require Basic Auth via requireAuth. GET returns the current settings as JSON. PUT accepts a JSON body with optional 'stt_language' (string) and 'tts_speed' (float) fields — omitted fields keep their current values. On PUT, persist to the settings file. Use the same mutex as the settings struct.

## Acceptance Criteria

GET /settings returns current values. PUT /settings with partial body updates only specified fields and persists to disk.
